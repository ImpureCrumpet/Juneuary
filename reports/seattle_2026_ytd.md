## Method & API Care

- This run uses cached observations from the local SQLite DB; the report itself makes **0 API calls**.
- Backfilling the 156-day window cost **2 Open-Meteo calls per city** (one weather, one air-quality), routed automatically across the `archive` (ERA5) and `/forecast` endpoints. Well under the free-tier daily limit.
- Re-runs use `fetch_weather.py --skip-existing` so we never re-pull dates we already have.
- Classification follows the three-tier model (primary / secondary constants / triggered events) defined in `scripts/classify.py`.
- Aberrations are days whose anomaly against the relevant monthly climate normal exceeds the threshold in `classify.py`. They are preserved as data but excluded from "propose new microseason" output and downgraded in primary-match confidence.
